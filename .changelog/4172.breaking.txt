Bump autoinstrumentation images: .NET 1.9.0 → 1.15.0, Java 2.20.1 → 2.27.0, Node.js 0.54.0 → 0.73.0

.NET Auto-Instrumentation (1.9.0 → 1.15.0):
- .NET 6 and .NET 7 runtime support removed; applications must run on .NET 8+
- `OTEL_DOTNET_AUTO_SQLCLIENT_SET_DBSTATEMENT_FOR_TEXT` and `OTEL_DOTNET_AUTO_ENTITYFRAMEWORKCORE_SET_DBSTATEMENT_FOR_TEXT` env vars removed (behavior is now always-on with sanitization)
- Kafka/RabbitMQ `otel.status_code` and `otel.status_description` tags replaced by native OpenTelemetry Status fields
- `Google.Protobuf`, `Grpc.Core.Api`, `Grpc.Net.Client`, `Grpc.Net.Common.dll` dependencies removed from distribution
- MongoDB instrumentation rewritten to bytecode; requires `MongoDB.Driver.Core` 2.7.0+
- `DOTNET_ADDITIONAL_DEPS` / `DOTNET_SHARED_STORE` assembly conflict workaround removed
- OpenTracing support and Zipkin exporter deprecated

Java Auto-Instrumentation (2.20.1 → 2.27.0):
- Spring Web `WebApplicationContextInstrumentation` moved from `spring-web` module to `spring-webmvc`; disabling via `otel.instrumentation.spring-web.enabled=false` no longer works, use `otel.instrumentation.spring-webmvc.enabled=false`
- AWS SDK 2.x span attributes renamed: `aws.bucket.name` → `aws.s3.bucket`, `aws.queue.url` → `aws.sqs.queue.url`, `aws.stream.name` → `aws.kinesis.stream_name`, `aws.table.name` → `aws.dynamodb.table_names`
- HTTP client URL query parameters now redacted by default in library instrumentations
- JDBC data source instrumentation, Finatra controller spans, and JSP compile spans disabled by default (v2.22.0); Play controller spans disabled by default (v2.23.0)
- Configuration properties deprecated (old names still work but will be removed): `*statement-sanitizer.enabled` → `*.query-sanitization.enabled`, `otel.instrumentation.common.database.*` → `otel.instrumentation.common.db.*`
- Logback `captureArguments` no longer captures message template; use `captureTemplate=true` explicitly

Node.js Auto-Instrumentation (0.54.0 → 0.73.0):
- Minimum Node.js version raised to 18.19.0+ or 20.6.0+ (Node.js 14 and 16 no longer supported)
- Upgraded to OpenTelemetry JS SDK 2.x; `BasicTracerProvider.addSpanProcessor()`, `MeterProvider.addMetricReader()` removed, `Resource` class eliminated in favor of `resourceFromAttributes()`
- `@opentelemetry/instrumentation-fastify` fully removed; migrate to `@fastify/otel`
- `@opentelemetry/instrumentation-redis-4` removed and consolidated into `@opentelemetry/instrumentation-redis`
- New instrumentations enabled by default: `instrumentation-runtime-node` (v0.58.0), `instrumentation-oracledb` (v0.59.0), `instrumentation-openai` (v0.66.0) — may produce unexpected telemetry and additional cost
- Programmatic config now correctly takes precedence over environment variables (was previously ignored when env vars were set)
- Environment resource detector now runs last, giving `OTEL_RESOURCE_ATTRIBUTES` / `OTEL_SERVICE_NAME` highest priority among detectors
